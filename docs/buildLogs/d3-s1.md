# Session-1

## Date: June 25

**Duration:** 90 minutes

**Goal:**  
Implement Rule of Three for HashMap and separate the implementation into `.h` and `.tpp` files.

**Problem Encountered:**  
HashMap required proper object copying support. Direct copying could result in shallow copy issues because HashMap internally stores buckets using a custom DynamicArray containing SingleLinkedList objects.

Template implementation separation also needed to be handled carefully because template definitions must be available during compilation.

**What I Tried:**  
- Added copy constructor for creating independent copies of HashMap objects.
- Implemented destructor for proper cleanup of stored key-value pairs.
- Added assignment operator overloading with self-assignment handling.
- Moved class declarations into `HashMap.h`.
- Moved template method definitions into `HashMap.tpp`.
- Used existing DynamicArray and SingleLinkedList copy mechanisms for deep copying buckets.

**Outcome:**  
Successfully implemented Rule of Three for the HashMap data structure.

HashMap objects now create independent copies instead of sharing internal memory.

Separated `.h` and `.tpp` files improved project structure and maintained template functionality.
